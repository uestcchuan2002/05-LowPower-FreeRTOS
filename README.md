# 基于 FreeRTOS 的低功耗 IoT 终端设计与验证

## 1. 项目简介

针对移动 IoT 设备续航与实时性难以兼顾的痛点，本项目基于 STM32F1 系列与 FreeRTOS 构建了一套高效的低功耗调度、多传感器采集与外设电源管理架构。项目采用软硬协同与模块化分层设计，成功实现了多任务并发、丰富的 UI 交互、极速唤醒与极限低功耗的完美平衡，非常适合应用于各类电池供电的物联网终端节点。
### 1.1 系统移植与部署必要条件
本低功耗方案采用模块化设计，可快速复现至其他硬件平台，但目标系统需满足以下核心要素：

* RTOS 调度支持 (FreeRTOS)
    **核心配置**：必须开启 configUSE_TICKLESS_IDLE 宏。
    **作用逻辑**：系统利用 FreeRTOS 的空闲任务（Idle Task）作为进入低功耗的入口。通过 RTOS 掌握的全局任务就绪状态，精确计算出系统可以“合法休眠”的最长时间，从而避免了传统轮询模式下频繁唤醒带来的功耗浪费。

* 硬件功耗等级切换能力 (MCU Power Modes)
模式要求：主控芯片需具备多种功耗梯度（如 Sleep、Stop、Standby 等）\
    **浅睡眠 (Sleep)**：仅关闭 CPU 内核，外设继续运行，用于高频交互场景 \
    **深睡眠 (Stop/Deepsleep)**：关闭绝大多数时钟树，仅保留外部中断和 RTC 唤醒，这是实现uA量级静态功耗的关键 \
    **恢复能力**：硬件必须支持在中断触发后，能快速切回全速运行频率（如从 HSI/HSE 重新倍频），保证系统的实时响应
  
* 独立时基与补偿外设 (RTC & Compensation) \
硬件依赖：需具备独立供电域的 RTC 外设，并建议配有外部低速晶振 (LSE, 32.768kHz) \
补偿机制：在主时钟（SysTick）因进入深休眠而停止期间，RTC 需持续运行 \
核心算法：唤醒后，系统读取 RTC 的计数差值，并调用 vTaskStepTick() 函数将遗失的系统心跳同步补回。这确保了软件定时器、延时函数在经历长达数分钟甚至数小时的休眠后，依然能保持逻辑上的连续性与准确性 

## 2. 核心特性与技术亮点

### 2.1 Tickless 深度休眠与 RTC 精准补偿

* **机制重构**：深度定制 FreeRTOS 的空闲任务（Idle Task），启用并重构了 Tickless 低功耗调度框架。当系统无就绪任务时，自动计算最长休眠时间并进入低功耗模式。
* **时钟接力**：利用 STM32F1 的硬件 RTC 配合外部低速晶振 (LSE) 与 VBAT 后备电源，在主频时钟停止、甚至主电源掉电时维持时间基准，并在唤醒后精准完成 FreeRTOS 的系统 Tick 补偿。
* **效果达成**：整机最低功耗电流可低至 **60uA**，显著延长设备续航。

### 2.2 动态外设门控与 GPIO 漏电封锁

* **引用计数框架**：在应用层（`app_power_gate`）搭建了基于“引用计数”的外设电源门控模块。系统运行时动态监测 MPU6050、DHT11 及 OLED 屏幕等外设的使用状态，自动切断闲置外设的电源供应。
* **IO 状态管理**：在进入休眠模式前，统一切换未使用 GPIO 的工作模式为**模拟输入态 (Analog)**，从硬件底层彻底切断由于上下拉电阻或悬空引脚导致的漏电路径，使待机静态功耗进一步降低 **30%**。

### 2.3 毫秒级多源唤醒与独立 UI 调度

* **中断安全交互**：结合 NVIC 机制，设计了按键与传感器双重 EXTI 中断唤醒机制。结合 FreeRTOS 任务通知，确保在唤醒瞬间高效、安全地进行上下文切换。
* **解耦的 UI 交互层**：针对屏幕显示构建了独立的 UI 任务（`ui_task`），包含完整的多级菜单框架（`ui_menu`），实现温湿度（DHT11）、姿态数据（MPU6050）、实时时钟（RTC）的可视化与低功耗状态的动态监测。

## 3. 技术栈

* **主控芯片**: STM32F1xx 系列 (基于 HAL 库底层驱动开发)
* **实时操作系统**: FreeRTOS (Tickless 模式定制、任务间通信、资源管理)
* **外设与传感器**: MPU6050 (六轴姿态)、DHT11 (温湿度)、OLED 显示屏
* **核心架构**: APP/UI/Core 三层解耦架构、动态电源控制算法、RTC 时钟补偿

## 4. 核心代码结构说明

项目采用高内聚、低耦合的分层目录结构，便于后期维护与功能扩展：

```text
├── APP/                      # 📦 应用业务层：核心业务逻辑与数据采集
│   ├── app_power_gate.c      # 核心：基于引用计数的动态电源门控管理
│   ├── app_rtc.c             # 核心：RTC 时间管理与低功耗时钟补偿逻辑
│   ├── app_dht11.c           # DHT11 温湿度传感器采集逻辑
│   ├── app_mpu6050.c         # MPU6050 六轴传感器姿态解算
│   ├── app_key.c             # 按键检测与唤醒事件处理
│   └── app_breathingLed.c    # 呼吸灯状态指示处理
│
├── UI/                       # 🖥️ 用户交互层：OLED 菜单与数据可视化
│   ├── ui_task.c             # UI 渲染刷新专属任务 (独立调度)
│   ├── ui_menu.c             # 多级菜单交互框架核心
│   ├── ui_LowPower.c         # 低功耗状态专属 UI 页面
│   ├── ui_DHT11.c            # 温湿度实时数据仪表盘界面
│   ├── ui_MPU6050.c          # 姿态数据可视化界面
│   └── ui_rtc.c              # 实时系统时钟展示界面
│
└── Core/Src/                 # ⚙️ 硬件底层与系统层：HAL库驱动与 RTOS
    ├── freertos.c            # FreeRTOS 任务创建、IPC 与调度器配置
    ├── main.c                # 系统时钟树配置、外设初始化与主入口
    ├── rtc.c / gpio.c ...    # STM32F1xx 外设底层初始化代码
    └── stm32f1xx_it.c        # 系统异常与中断服务函数 (EXTI 唤醒等)

```

## 5. 快速上手与验证

1. **环境准备**：推荐使用 STM32CubeIDE 或 Keil MDK (MDK-ARM) 导入工程。
2. **硬件连接**：根据 `gpio.c` 与相关外设配置连接 MPU6050、DHT11 及 OLED。确保纽扣电池接入 VBAT 引脚。
3. **现象验证**：
* 系统上电启动后，通过实体按键进入 OLED 菜单，可实时查看各传感器读数与当前系统时间。
* 无操作一段时间后，系统关闭屏幕及非必要传感器供电，自动进入深度休眠，此时功耗仪读数应降至微安（uA）级别。
* 触发外部按键，系统瞬间点亮恢复工作状态，时间不丢失，实现无缝切换。
